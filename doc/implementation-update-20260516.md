# websqlitebrowser 구현 반영 문서 (2026-05-16)

## 개요
본 문서는 최근까지 반영된 기능/품질/보안/호환성 개선 사항을 정리한다.

## 1) Data Explorer 개선
- 최상위 디렉토리(파일시스템 루트)까지 상위 이동 가능하도록 처리.
- `../` 항목 기반 상위 이동 UX 유지.
- 권한 이슈(`PermissionError`) 발생 시 403으로 구분하고, 프론트 Output 로그에 명확히 표시.
- 권한 거부 상황에서도 가능한 범위의 부분 결과를 반환하도록 보완.

## 2) `directory_stats()` 성능 최적화
- 기존: 하위 디렉토리를 재귀 스캔(`rglob`)하여 느린 동작 가능.
- 변경: 현재 디렉토리의 직접 자식만 통계 집계(O(n)).
- 파일 크기 집계 시 권한 오류 항목은 안전하게 건너뛰도록 예외 처리 강화.
- 심볼릭 링크 추적을 비활성화하여 성능/안전성 확보.

## 3) SQL 실행 안정성/성능
- 읽기 전용 SQL 정책 강화(SELECT/WITH/PRAGMA/EXPLAIN 기반 검증).
- 멀티 스테이트먼트 실행 지원 및 결과 분리 처리.
- Oracle `ROWNUM` 패턴 호환 변환 지원.
- 캐시/페이지네이션/메트릭 수집(요청 시간, 쿼리 시간, 캐시 여부) 반영.
- API 타임아웃/재시도(지수 백오프) 추가.

## 4) 보안 강화
- SQL 인젝션 방어 강화:
  - 위험 패턴 차단(ATTACH/DETACH/VACUUM 등).
  - 쿼리 복잡도 제한(JOIN/서브쿼리 개수 제한).
- 경로 검증 강화:
  - null byte/비정상 입력 방어.
  - 경로 정규화 및 허용 범위 검증.
  - 심볼릭 링크 대상 범위 이탈 방어.
- 설정 보안:
  - LLM 토큰 저장 시 암호화 처리.
  - API 응답에서 토큰 마스킹(`***`).
- 민감 정보 마스킹:
  - 경로/토큰/이메일 패턴을 응답 텍스트에서 마스킹.

## 5) UI/브라우저 호환성
- Explorer 리스트 렌더링에서 Edge 호환성 이슈 대응:
  - 일부 Grid 기반 레이아웃을 Flex 동작 중심으로 보강.
- OS/디바이스별 스케일 분기:
  - 모바일: 기존 크기 유지.
  - Windows 데스크톱: Data Explorer 항목(헤더/행/텍스트) 1.5배 확대.

## 6) 테스트 상태
- Django 단위 테스트 전체 통과.
- Playwright E2E 테스트 전체 통과(23 passed 기준 확인).

## 7) 현재 상태 요약
- 사용자 요청 기반 핵심 항목(최상위 탐색, 권한 안내, 성능 병목 제거, 보안 강화, Edge/Windows 표시 개선)이 반영 완료.
- 다음 확장 후보:
  - `config.yml` 기반 설정 외부화.
  - 토큰 저장 암호화 고도화(전용 암호화 라이브러리/KMS).
  - 감사 로깅 및 운영 정책 세분화.
