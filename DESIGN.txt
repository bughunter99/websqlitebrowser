websqlitebrowser 프롬프팅 가이드
===================================

이 문서는 websqlitebrowser 프로젝트를 AI에게 설명할 때 필요한 모든 요구사항을 정리한 것입니다.
새로운 개발자나 AI에게 이 프로젝트의 전체 범위를 한 번에 전달하기 위한 종합 프롬프트입니다.

---

프로젝트 개요 및 핵심 요구사항
==============================

프로젝트 목표
-----------
Django + JavaScript로 SQLite 파일을 웹에서 탐색하고 조회하는 브라우저 도구를 만든다.
- 서버 디렉토리 내 SQLite 파일 탐색
- 테이블/데이터 조회
- SQL 직접 실행
- LLM 기반 자연어 질의응답 지원
- Windows/Linux 경로 처리 지원

개발 환경
---------
- 백엔드: Django (Python)
- 프론트엔드: Vanilla JavaScript + HTML5 + CSS
- DB: SQLite3
- 가상환경: `workon v1` (Python virtualenv)

주요 UI/UX 요구사항
===================

1. 전체 레이아웃
-----------
- 좌측: 고정 사이드바 (너비: 240-304px, 사이드바 resizer로 조정 가능)
- 우측: 메인 작업 영역
- 전체 색상 테마: Windows XP 클래식 스타일 (회색 배경 #c0c0c0, 타이틀바 파란색)

2. 좌측 사이드바 구조
-------------------
Activity Bar (아래쪽 고정)
- 3개 탭: Data Explorer, Chat, Setting
- 탭 전환 시 상단 콘텐츠 영역만 변경 (탭이 이동하지 않음)
- 텝 사이사이의 설명 텍스트 제거 (예: "Browse repository")

상단 콘텐츠 영역
- 현재 선택된 Activity의 내용 표시
- 스크롤 가능

3. Data Explorer
---------------
목표: Windows 파일 탐색기 또는 FileZilla 클라이언트처럼 보이기

기능 요구사항:
- 서버 repository 폴더 아래 디렉토리/파일 탐색
- 파일 목록은 얇은 행 높이로 최소 20개 이상 표시 가능하게
- 각 행: 파일명 | 용량 | 마지막 수정 시간 (YYYYMMDD HH:MM:SS 형식)
- 각 항목 맨 앞에 상위 폴더 ../ (현재 폴더에서도 최상위까지 올라갈 수 있게)
- 파일명이 길면 뒤를 ..로 자르기 (ellipsis)
- 필요없는 라벨 제거:
  - "Activity Sidebar" 라벨
  - "[DB]" 아이콘 표시
  - 선택 안내 메시지 ("선택된 SQLite 파일이 없습니다" 등)

키보드 네비게이션:
- 화살표 Up/Down: 현재 항목 위/아래로 이동 (페이지 스크롤 아님)
- Page Up/Down: 목록 스크롤
- Home: 맨 처음 항목
- End: 맨 마지막 항목

디렉토리 구조 준비:
- repository/ 폴더에 SQLite 파일 100+ 개 배치 (파일명: db1.sqlite, db2.sqlite, ... db100.sqlite)
- 서브폴더 구조도 추가 (여러 단계로 이동 가능하도록)

상태바:
- Data Explorer 아래에 Windows Vista/7 스타일의 상태바 추가
- 표시 정보: 현재 폴더 경로, 폴더 개수, 파일 개수, 전체 용량, 디스크 사용률

4. 메인 패널 (우측)
------------------
상단 영역
- 현재 열린 DB 정보 표시 (경로 등)
- 리로드 버튼 등 필요한 액션만 표시

탭 영역
1. Query 탭 (항상 첫 번째)
   - 상단: SQL 입력 영역 (고정폭 폰트 "Courier New", 크기 12px)
   - 분할선 (마우스 드래그로 조정 가능)
   - 하단: 결과 그리드

2. DDL 탭 (필요시)
   - 테이블의 컬럼 정의와 인덱스 정보 표시
   - "컬럼 정의와 인덱스, 원본 CREATE SQL을 확인할 수 있습니다." 같은 설명 제거
   - 스크롤바 추가 (내용이 길 경우)
   - CREATE TABLE 구문만 보여주기

3. 테이블 탭들 (테이블마다 하나씩)
   - 탭 이름: 테이블 이름
   - 내용: SELECT * FROM table LIMIT N 결과를 그리드로 표시

최하단: Output 창
- Visual Studio 6.0 스타일의 Output 창
- 로그 내용: 디렉토리 이동, 쿼리 실행, 에러 메시지 등
- 각 라인 앞에 시간 표시 (YYYYMMDD HH:MM:SS 형식)
- 우측 상단에 "Auto Hide" 버튼 (토글 가능, 접었다 펼칠 수 있음)
- 텍스트 크기: 12px (고정)
- Clear 버튼

그리드(테이블) 요구사항
======================

기본 스타일
-----------
- 배경: 흰색, 텍스트: 검은색
- 헤더 배경: 회색 (#d4d0c8-#c0bbb1 그라데이션)
- 테두리: 1px dotted #c7c7c7 (셀 사이의 얇은 점선)
- 고정 행 높이: 22px
- 긴 텍스트는 말줄임표(...)로 표시 (넓은 줄 생성 금지)

열 기능
-------
- 초기 열 너비: 헤더 텍스트 길이 기반 (공식: Math.ceil(text.length * 8) + 24, 최소 72px, 최대 320px)
- 행 번호 열 (#): 36px 너비, 좌측 고정
- 열 리사이징: 마우스로 헤더 경계 드래그 가능
- 열 정렬: 헤더 클릭으로 오름차순/내림차순 토글 (정렬 표시: ▲▼)

셀 선택 및 상호작용
------------------
- 셀 단일 선택 가능
- 다중 선택 가능 (Ctrl+Click)
- 범위 선택 (Shift+Click)
- Ctrl+C: 선택된 셀 내용 클립보드 복사
- 우클릭 컨텍스트 메뉴:
  - 셀 복사
  - 선택 범위 복사
  - 행 전체 복사
  - 선택 해제
  - 행 전체 선택

대용량 데이터 처리
-----------------
- 행이 1000개 이상인 경우 가상화(virtualization) 적용
- 부드러운 스크롤 성능 필요
- 무한 스크롤 또는 페이지네이션 지원

SQL 결과 표시
-----------
- 여러 SELECT 쿼리 실행 시 각각 별도 탭으로 표시 (result1, result2, ...)
- 쿼리 메타데이터: "실행 시간: YYYY-MM-DD HH:MM:SS, Fetched: N rows"

SQL 에디터 및 쿼리 실행
=======================

SQL 에디터
---------
- 폰트: "Courier New" (또는 monospace 계열), 크기 12px
- Syntax Highlighting: 기본 키워드는 파란색 (SELECT, FROM, WHERE 등)
- F9 키: 쿼리 실행
- Ctrl+Enter: 쿼리 실행

쿼리 실행
--------
- Oracle 호환 문법 지원 (예: SELECT count(*) FROM abcd WHERE rownum < 11;)
- SQLite 기본 쿼리도 정상 실행
- 읽기 전용 (SELECT, PRAGMA, WITH 등만 허용)
- 실행 결과와 에러 메시지를 Output 창에 로깅

기술 구현 세부사항
==================

백엔드 (Django)
--------------
모듈 분리 계획:
1. file_browser.py: 허용 경로 탐색, 디렉토리/파일 정보 조회
2. sqlite_service.py: DB 연결, 테이블 목록, 스키마, 쿼리 실행
3. settings_service.py: LLM 설정 저장/조회
4. chat_service.py: 프롬프트 생성, SQL 생성, 응답 반환

API 엔드포인트:
- 파일 탐색 API
- 테이블 목록 조회 API
- 테이블 메타데이터 (DDL, 스키마) API
- SQL 실행 API (여러 쿼리 파싱 지원)
- 설정 저장/조회 API
- Chat API

주의사항:
- Windows/Linux 경로 처리: pathlib.Path 사용
- 허용 루트 경로 제한 (설정 파일 또는 환경변수)
- SQLite 읽기 실패/잠금 예외 처리
- 쿼리 결과 타임아웃/row limit (예: 10000행)

프론트엔드 (JavaScript)
---------------------
모듈 분리 계획:
1. app.core.js: 그리드 렌더링, 정렬, 리사이징, 선택
2. app.events.js: UI 이벤트 처리 (클릭, 키보드, 컨텍스트 메뉴)
3. app.css: 모든 스타일 (Windows XP 클래식 테마)

핵심 함수들:
- renderTable(): 정규 HTML 테이블 렌더링 (행 < 1000)
- renderVirtualizedTable(): 가상화 그리드 (행 >= 1000)
- attachGridInteractions(): 선택, 정렬, 리사이징, 컨텍스트 메뉴
- copySelectedCells(): 클립보드 복사 (SQL 에디터 및 그리드)
- outputLog(): Output 창에 로그 추가
- setOutputAutoHide(): Output 자동 숨김 토글

상태 관리:
- WeakMap 기반 그리드별 상태 추적 (정렬 정보 등)

CSS 색상 및 치수
---------------
색상 팔레트 (Windows XP 클래식):
- 배경: #c0c0c0
- 표면: #d4d0c8
- 밝은 표면: #ffffff
- 강한 선: #5a5956
- 텍스트: #111111
- 강조: #0a246a
- 타이틀 배경: linear-gradient(180deg, #0a246a, #4d7cc6)

주요 치수:
- 행 높이: 22px
- 행 번호 열 너비: 36px
- 데이터 열 너비: 72-320px (동적)
- 사이드바 너비: 240-304px (조정 가능)
- 폰트 크기: 본문 12px, 작은 텍스트 10px, 입력창 12px

제약사항 및 특수 요구사항
========================

1. OS 구분: 개발 환경은 Termux + Ubuntu (proot-distro), 배포 시 Windows/Linux 지원
2. 환경 활성화: 항상 `workon v1` 후 명령 실행
3. 포트: Django 기본 포트 사용
4. DB 파일: repository 폴더 내에만 접근 (보안)
5. 캐시 버전: CSS/JS 변경 시 index.html의 버전 번호 증가 필요
6. Responsive: 브라우저 크기 변경 시 UI도 함께 리사이징
7. 모바일 vs 데스크톱: 폰에서는 글자 크기 작게, Windows에서는 1.2배 크게
